# CLAUDE.md — Авто-Рилс

Агентские инварианты. Читается каждую сессию. Держать коротким и высокосигнальным.

## Что строим

CLI-инструмент: длинное горизонтальное видео (talking-head, статичная камера) → набор вертикальных рилсов 9:16 с выжженными субтитрами и кликбейт-заголовками. **Полностью автоматически.** Облако выбирает моменты (текст), локаль рендерит (ffmpeg).

Полная архитектура — `PROJECT_GUIDE.md`. Ядро (выбор моментов) — `R0_SPEC.md`. План реализации — `PLAN.md`. Раскладка — `PROJECT_STRUCTURE.md`.

## Несущая граница: ОБЛАКО / ЛОКАЛЬ

Делит весь код на два тира. **Видео между тирами не передаётся** — облако работает только с текстом (аудио → транскрипт → манифест), исходник живёт на локали, где и рендерится. Это выражено в структуре каталогов (`src/cloud/` vs `src/local/`), не только в соглашении.

## Жёсткие инварианты (не нарушать)

1. **Determinism-first.** LLM только ранжирует и предлагает кандидатов. Финальные границы реза, отбор, валидацию ставит детерминированный код. Скоры от LLM — подсказка для сортировки, не источник истины.
2. **Grounding обязателен.** Каждое поле от LLM (`start/end/title/...`) привязано к фрагменту транскрипта. Ничего не выдумывается из подразумеваемого.
3. **Пустой массив — валидный результат.** `segments: []` («хороших моментов нет») — ожидаемый исход, не ошибка. Не падать, не ретраить ради непустоты.
4. **Идемпотентность by design.** Транскрипт кэшируется по хэшу аудио; сегменты — по ключу прогона (хэш source+preset+версия рубрики). Повторный запуск не плодит дубли и не перетранскрибирует. (Та же ловушка, что баг размножения задач в Meeting→Tasks — не повторять.)
5. **Two-phase.** Phase 1 (выбор+нарезка) гонит автоматически до `awaiting_review`. Phase 2 (финальный рендер + выдача) — только по approve. В MVP-0 review нет → auto-approve, но фазы разделены в коде.
6. **Чек-флаги детерминированы.** `too_long`/`too_short`/`no_hook`/`cut_midword` ставит код, не модель.

## Технические ограничения

- **Только бесплатные API.** LLM — Groq (Qwen) → OpenRouter failover. Whisper — Groq или локальный faster-whisper. Узкое место — TPM на R0 (6K Groq free): транскрипт сжимать (sentence-level) + чанкить + троттлить по `x-ratelimit` заголовкам + prompt-cache на рубрику.
- **Reasoning-выхлоп бьёт по TPM.** qwen3 — reasoning-модель; reasoning раздувает выходные токены и упирает в 6K TPM (перемежающийся 413). Для структурной выборки по чёткой рубрике reasoning не нужен → `reasoning_effort=none`. M1-чанкинг: token-бюджет = вход + reasoning-выхлоп, не только вход.
- **Железо:** 8 ГБ GPU / 32 ГБ ОЗУ. Qwen 32B локально НЕ влезает → R0 через API. Рендер — ffmpeg локально (NVENC где можно).
- **Промпты:** system на английском, вывод на русском (валидированный паттерн: крепче grounding на русской речи).

## Кодовые конвенции

- Python 3.11+, FastAPI (тир оркестрации появляется в M1), SQLAlchemy + SQLite (dev) → PostgreSQL (prod-debt, не для MVP).
- Манифест — единственный контракт между тирами. Схема в `src/core/models.py` (Pydantic). Любое изменение схемы — через неё.
- Провайдер-абстракция (`src/cloud/providers.py`): Groq/OpenRouter/локаль за одним интерфейсом. Round-robin + бэкофф по `retry-after`.
- Конфиг вынесен в YAML (`config/`), не хардкод: пресеты длины, пороги score, размеры чанков, профиль кропа.

## Дисциплина разработки

- **TDD.** Сначала тест на детерминированный слой (валидаторы, дедуп, snap границ, парсинг манифеста), потом код. LLM-вызовы мокаются фикстурами реальных ответов.
- **Стоп-чек на R0.** Прежде чем строить рендер — прогнать R0 на реальном 46-мин транскрипте и глазами проверить выборку. Если выборка плохая — НЕ идти дальше.
- **Инкрементально.** Каждый milestone (`PLAN.md`) даёт реальный выход до начала следующего. Не пре-архитектурить под то, чего ещё нет.
- **Минимализм.** Простейшее, что работает. Сложность откладывать явно, не закладывать заранее.

## Чего НЕ делать (явные анти-цели MVP)

- НЕ писать трекинг / active-speaker detection. Кроп — статичный прямоугольник из профиля сетапа.
- НЕ рендерить в облаке. Рендер только локальный ffmpeg.
- НЕ гнать видео через API. В облако уходит только аудио и текст.
- НЕ строить yt-dlp приём, SMM-стык, RAG в MVP-0 — это M2+.
- НЕ оптимизировать преждевременно (PostgreSQL, мультипользовательность) — отложенный долг.
